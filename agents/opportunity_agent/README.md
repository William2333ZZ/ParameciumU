# 商机智能体（运营商行业商客业务）

面向运营商行业 **商客经理** 的日常工作智能体，聚焦“商机录入 / 商机查询 / 商机清单”三类高频任务。

> 说明：本 Agent 目前提供工作流与字段规范、对话指令与输出格式约束；
> 若要实现「多渠道语音录入」等能力，需要在 Gateway 侧接入语音转文字（ASR）与渠道连接器（飞书/企微/电话录音等）节点，再由本 Agent 做标准化、去重与状态流转。

## 主要能力

### 1) 商机录入

- 多渠道输入：文字/粘贴/转写文本（来自语音、IM、邮件等）。
- 标准化：自动抽取并补齐关键字段，输出“可入库”的结构化商机。
- 状态跟踪：按商机全流程阶段维护状态、下一步动作、预计关闭时间等。
- 智能处理：
  - 去重提示（同企业/同门店/同联系人/同地址相似）
  - 完整性校验（缺字段时列出最少追问）
  - 风险/优先级建议（基于金额、时限、客户体量、竞争态势）

### 2) 商机查询

- 模糊查询：按照一个或多个条件，返回某场景的商机数量（以及可选的Top列表）。
- 定向查询：按某个企业/商铺（名称/统一社会信用代码/地址/联系人/电话）检索其商机信息。

### 3) 商机清单

- 按用户要求对“用户名下商机”做多重筛选/排序/分组，生成可复制的清单（Markdown 表格/CSV）。

## 商机字段规范（建议）

最小可用字段（必填/强建议）：

- 归属：owner（商客经理）、组织（地市/区县/网格）
- 客户：customerName、customerType（企业/商铺/园区…）、industry
- 联系：contactName、contactPhone（可脱敏展示）、contactRole
- 位置：address（可拆分省市区/经纬度如有）
- 需求：scenario（场景）、products（意向产品：专线/云/物联网/固话/宽带/短信…）
- 规模：expectedAmount（月费/年费/一次性）、expectedLines/term
- 阶段：stage（线索/初步接触/方案中/报价/合同中/交付中/赢单/输单）
- 计划：nextAction、nextActionDate、expectedCloseDate
- 来源：sourceChannel（电话/走访/转介绍/活动/线上…）、sourceDetail
- 备注：notes

## 推荐对话指令

- 「录入商机：……」（直接粘贴线索文本/转写文本）
- 「把这段语音转写的线索标准化成商机」
- 「查询：我名下本月‘方案中’阶段、行业=餐饮的商机数量」
- 「查一下某某商贸有限公司的商机」
- 「生成清单：按优先级排序，过滤掉预计金额<5000/月的」

## 数据与工具

本仓库未默认提供商机数据库/检索服务。本 Agent 采用“结构化输出 + 可对接”的方式：

- 入库：输出 JSON（可被上游系统写入 CRM/自建库）
- 查询/清单：需要上游系统把命中的商机记录（JSON 数组）提供给本 Agent 做筛选、统计与呈现

如需我继续把它落地成可用服务，请你补充：

1) 你们现有 CRM/商机系统：字段、接口（OpenAPI/数据库/Excel 导入）
2) 语音来源渠道：飞书/企微/电话录音/钉钉等
3) 商机流程阶段与口径：你们内部的标准阶段/赢单输单原因字典
