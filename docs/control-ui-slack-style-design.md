# Control UI 改版设计：Slack / 飞书 / 微信形态

本文档描述将 Control UI 改版为「以聊天和群聊为核心」的形态，突出：**智能体注册与上线**、**拉群与聊天**，整体交互参考 Slack、飞书、微信。

---

## 一、产品目标（改版后）

- **主心智**：把 monoU 控制台当作「和智能体/群聊沟通」的 IM 工作台，而不是「先选 Tab 再选功能」的控制面板。
- **核心任务**：
  1. **智能体注册与上线**：能看到哪些 Agent 已连接、在线，并一键发起单聊。
  2. **拉群与聊天**：有「私聊」（与单个 Agent）和「群聊」（多成员/多 Agent 或来自飞书等接入的群），主区域始终是当前会话的聊天界面。
- **次要能力**：拓扑、Cron、浏览器、设置等仍保留，但收起到侧栏次要入口，不抢主流程。

---

## 二、参考形态简述

| 产品   | 左侧                     | 中间                 | 右侧/主区           |
|--------|--------------------------|----------------------|---------------------|
| Slack  | 工作区名 + 频道/私信列表 | 当前频道/私信消息流  | 有时为线程/详情     |
| 飞书   | 工作台 + 会话/群组列表   | 当前会话消息 + 输入  | 可选侧边详情        |
| 微信   | 仅会话列表（置顶+最近）  | 当前聊天消息 + 输入  | —                   |

共同点：**左侧是会话/联系人列表，主区域是当前会话的聊天内容**。进入产品后默认就是「选一个会话 → 看消息、发消息」。

## 三、整体布局（三栏）

```
┌─────────────────────────────────────────────────────────────────────────┐
│  [●] monoU                                    [设置] [断开] (可选)       │  ← 顶栏（可选，或合并到左栏）
├─────────────┬───────────────────────────┬───────────────────────────────┤
│             │                           │                               │
│  一级导航   │    会话列表（二级）        │    主内容区                    │
│  (窄)       │    (中宽)                 │    (主聊天区)                  │
│             │                           │                               │
│  消息       │  私聊                      │  ┌─ 会话标题（Agent/群名）    │
│  智能体     │    · .u                    │  │  可选：成员、会话信息      │
│  ─────      │    · pilot                 │  ├────────────────────────────│
│  群聊       │  群聊                      │  │  消息流                    │
│  全部会话   │    · 飞书-产品组           │  │  ...                       │
│  ─────      │    · + 新建群聊            │  │                            │
│  拓扑       │  全部会话(最近)            │  ├────────────────────────────│
│  Cron       │    · agent:.u:s-xxx       │  │  [输入框]  [发送]           │
│  浏览器     │    · ...                   │  └────────────────────────────┘
│  设置       │                           │                               │
│             │                           │                               │
└─────────────┴───────────────────────────┴───────────────────────────────┘
```

- **左栏（一级导航）**：图标 + 文字或仅图标，可折叠。
  - **消息**（默认选中）：对应「聊天主形态」，右侧中栏即「会话列表」，再右侧为「当前会话聊天」。
  - **智能体**：当前「拓扑」的另一种入口，强调「发现/管理智能体」，可展示在线 Agent 列表并支持「发起对话」。
  - **群聊**：可单独作为一级，或与「消息」合并（消息下再分子列表：私聊 / 群聊 / 全部会话）。
  - **拓扑 / Cron / 浏览器 / 设置**：次要入口，点击后主内容区切换为对应面板（拓扑图、Cron 列表、浏览器、设置），与现有实现一致。

- **中栏（会话列表，仅当「消息」或「群聊」激活时显示）**：
  - **私聊**：来自 `node.list` 的每个在线 Agent 一行，显示 `agentId`、在线状态；点击即 `openChat(agentId)`，主区加载该 Agent 的默认/主会话。
  - **群聊**：来自 `sessions.list` 中可识别为「群」的会话（如 `channel` 含 group、或 `sessionKey` 为 `connector:feishu:chat:xxx` 等）；支持「+ 新建群聊」入口（若后端支持则创建群 session，否则占位）。
  - **全部会话**：即现有 `sessions.list` 按 `updatedAt` 排序，展示最近会话，点击在主区打开。

- **右栏（主内容区）**：
  - 当选中「某私聊/某群聊/某会话」时：顶部会话标题（Agent 名或 `displayName` 或 sessionKey 短显），下方为**消息流 + 输入框**（即现有 ChatPanel 能力）。
  - 当未选会话或选「新建群聊」时：欢迎页或引导「从左侧选一个会话」/「选择要拉群的智能体」。
  - 当左栏选「拓扑 / Cron / 浏览器 / 设置」时：主内容区显示对应全屏面板，可隐藏中栏或保留中栏占位。

## 四、信息架构与数据对应

| 界面概念     | 数据来源 / 行为 |
|--------------|------------------|
| 私聊列表     | `node.list` → 各 node 下 `agents[]`，去重后按 agentId 列表展示；点击 → `openChat(agentId)`，sessionKey 可选默认（如 main）或由后端解析。 |
| 群聊列表     | `sessions.list` 中 `channel` 为群或 sessionKey 匹配 `connector:*:chat:*` 等；「+ 新建群聊」→ 占位或调用后续「建群」API。 |
| 全部会话     | `sessions.list`，按 `updatedAt` 排序；点击 → `openChat(agentIdFromKey(s.key), s.key)`。 |
| 当前聊天主区 | 与现有 ChatPanel 一致：`initialAgentId` + `initialSessionKey`，消息流、输入框、工具调用展示。 |
| 智能体入口   | 与拓扑/Overview 同源：`node.list` + `connectors`，可列表/图切换，每 Agent 提供「发消息」→ 跳转消息并 openChat。 |

## 五、关键交互流程

### 5.1 进入后首屏

- 连接成功后，默认进入「消息」视图：左栏选「消息」，中栏展示「私聊 + 群聊 + 全部会话」，右栏若无选中会话则显示欢迎页（如「从左侧选一个智能体开始对话」或默认选中第一个在线 Agent 的会话）。

### 5.2 和某个智能体单聊

- 在「私聊」里点某 Agent → 主区打开与该 Agent 的对话（当前即 `openChat(agentId)`）。
- 或在「智能体」里点某 Agent 的「发消息」→ 同上。

### 5.3 进群聊 / 拉群

- **进群**：在「群聊」列表点已有群会话 → 主区打开该 sessionKey 的聊天。
- **拉群**：点「+ 新建群聊」→ 若当前有「建群」能力则弹出选 Agent/选成员；否则先做占位（例如创建 `agent:.u:group-<ts>` 这类 session，在 UI 上标为「群聊」）。

### 5.4 从「全部会话」继续聊

- 与现有 SessionsPanel 一致：点某条会话 → `onOpenSession(agentId, sessionKey)`，主区加载该会话。

### 5.5 拓扑 / Cron / 设置

- 左栏点「拓扑」→ 主内容区全屏显示拓扑（列表/图），可隐藏中栏或保留窄中栏。
- 左栏点「Cron」「浏览器」「设置」→ 主区显示对应面板；从拓扑/会话点「对话」仍可调用 `openChat` 并切回「消息」+ 选中对应会话。

## 六、与现有实现的对应关系

| 现有                 | 改版后                                       |
|----------------------|----------------------------------------------|
| 侧栏 Tab：对话       | 左栏「消息」默认；主区 = 聊天                |
| 侧栏 Tab：拓扑       | 左栏「智能体」或「拓扑」；主区 = 拓扑面板    |
| 侧栏 Tab：会话       | 中栏「全部会话」区块，或合并进「消息」视图  |
| 侧栏 Tab：Cron/浏览器/设置 | 左栏保留，主区切换对应面板            |
| ChatPanel            | 主内容区「当前会话」区域，逻辑不变           |
| SessionsPanel        | 中栏「全部会话」列表 + 点击打开会话          |
| OverviewPanel 的 Agent 列表 | 中栏「私聊」列表 + 智能体页的列表/图   |
| openChat(agentId, sessionKey?) | 不变，从私聊/群聊/全部会话点击统一调用 |

## 七、形态小结（可直接落地的结构）

1. **三栏布局**：左 = 一级导航（消息 / 智能体 / 群聊 / 拓扑 / Cron / 浏览器 / 设置），中 = 会话列表（私聊 / 群聊 / 全部会话），右 = 主内容（当前聊天或其它面板）。
2. **默认首屏**：连接后即「消息」视图 + 会话列表 + 欢迎或默认会话，突出「聊天」。
3. **智能体上线**：通过「私聊」列表和「智能体」入口展示在线 Agent，点击即聊。
4. **拉群与群聊**：「群聊」列表展示已有群会话；「+ 新建群聊」占位或接后续建群能力。
5. **路由/状态**：可沿用内存 state（当前选中的 tab、会话、agentId、sessionKey）；若引入路由，建议 `#/message`、`#/message?agentId=xxx&sessionKey=xxx`、`#/topology` 等，便于书签与分享。

此设计在保留现有 Gateway/会话/拓扑能力的前提下，把界面形态收敛为 Slack/飞书/微信式的「以聊天和群聊为中心」的形态，便于后续迭代多 Agent 群聊与更多 Connector 的群会话展示。
